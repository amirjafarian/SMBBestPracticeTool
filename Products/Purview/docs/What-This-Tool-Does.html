What this script does in your tenant

1 Run modes — how the script behaves when you launch it

Apply Default

No switch needed

Every change is applied to the tenant. The script asks for confirmation once at the start (skip with -NonInteractive).

Preview -WhatIf

Read-only

Walks through every step and prints what would happen, without writing anything to the tenant. Use this for change-approval review.

Per-action prompt -Confirm

Interactive

Pauses before each individual change. Useful when migrating an existing tenant and you want to inspect each step.

Unattended -NonInteractive

CI / automation

Skips the preflight confirmation. Combined with -AutoInstallModules the script runs end-to-end with no human input. Tenant-identity guard still aborts on mismatch before any write.

2 Five tasks — what the script touches, in order

1. Tenant settings Default

Foundational on/off switches: unified audit log, AIP×SPO integration, PDF labels.

Container labels default-on (opt out with -SkipContainerLabels) · -EnablePremiumAudit

2. Sensitivity labels Default

Creates 3 label groups (Public is standalone) & 6 assignable children, orders them, applies encryption to Highly Confidential and Confidential\Trusted People, and publishes the user-facing subset.

-SkipLabels to opt out

3. DLP Default

Two policies block external sharing of Confidential / Highly Confidential content — one for Exchange, one for SharePoint+OneDrive.

-SkipDLP to opt out · created in simulation by default

4. Retention Opt-in

7-year retain-then-delete on Exchange mailboxes. Off by default — you must pass -ApplyRetention after confirming the duration matches the customer's vertical.

Destructive deletes mail older than 7 years

5. AI governance Auto-detected

Blocks Microsoft 365 Copilot from grounding on Highly Confidential content. Default ON for E5 / Purview Suite tenants; auto-skipped on Business Premium.

-SkipAIControls · -BPOnly hard-skip

4 Sensitivity labels created

Ten labels — Public plus three label groups (General, Confidential, Highly Confidential), each with two assignable children — ordered from least sensitive (top) to most sensitive (bottom). Labels are identified/created by Microsoft's stable internal signature so the same taxonomy lands correctly on any tenant, locale, or label scheme (classic or modern). Encryption is intentionally only applied to Highly Confidential\All Employees (Template), Highly Confidential\Specific People and Confidential\Trusted People (both UserDefined); every other label carries footer/watermark marking only, which is the SMB-friendly profile that minimises breakage with third-party tooling.

The Applies to line under each label reflects the scope the script sets. The four published labels (Public, General — Anyone (unrestricted), Confidential — All Employees, Highly Confidential — All Employees) are the only ones shown in the Office Sensitivity picker; the other three (General \ All Employees (unrestricted), Confidential \ Trusted People, Highly Confidential \ Specific People) are created so DLP can match on them but stay off the client UI. Of the published set, General — Anyone (unrestricted), Confidential — All Employees and Highly Confidential — All Employees ship with Files · Emails · Groups & sites so they appear in the Sensitivity picker when a user creates a new Team, M365 Group or SharePoint site; every other label starts at Files · Emails only. Pass -SkipContainerLabels to drop the Groups & sites bits (labels still ship with Files & Emails). You can widen any label later in the Purview portal without losing the deployment baseline — adoption uses UNION-not-replace, so any scope you add by hand is preserved on the next re-run.

The colored dot beside each label name matches the per-label Color the script writes to the label (with drift correction on re-run). Operators see this same color in the Purview portal and end users see it on the Sensitivity button in Word, Excel, PowerPoint and Outlook.

i
Rights bundle: by default encrypted labels grant Microsoft's Reviewer bundle (View, Edit, Save, Reply, Reply All, Forward). Highly Confidential — All Employees overrides this with the wider Co-Author bundle (adds Copy, Print, Allow Macros — needed for Office co-authoring and any third-party tool that reads doc metadata via the Office object model). Each label can declare its own EncryptionRightsDefinitions in PurviewConfig.psd1; labels without an override fall back to the global default. Edit either and validate in a pilot tenant before rolling out.
!
Visual content marking is off by default. Footer text and the "HIGHLY CONFIDENTIAL" watermark are configured on the labels but only render once you flip EnableContentMarking = $true in PurviewConfig.psd1. This lets you stage the rollout without surprising end-users.

5 Label publishing — what end users see

All ten labels are created in the tenant. Only the labels below are published to end users via the SMBTool - Default Label Policy. Sub-label names are auto-expanded to include their parent so the Purview hierarchy stays valid.

SettingValueWhat it means for end users
Published labels Public · General / Anyone (unrestricted) · Confidential / All Employees · Highly Confidential / All Employees The four labels that appear in the Office "Sensitivity" picker for every licensed user.
Default for documentsConfidential / All Employees Every new Word, Excel and PowerPoint document is automatically tagged so it carries the Confidential footer.
Default for emailGeneral / Anyone (unrestricted) Every new Outlook message is pre-tagged General so routine internal mail doesn't trigger DLP. General itself is a non-applicable label group.
Inherit label from attachments ONAttachmentAction = 'Automatic' If an outgoing email has a labelled attachment with a higher-priority label than the email's current label, Outlook silently upgrades the email to match — no prompt.
Mandatory labelling ON$true Office prompts the user to pick a sensitivity label before saving a new document or sending a new email. Closes the untagged-content hole that breaks DLP, auto-classification and Copilot exclusions.
Downgrade justification ON$true Users must give a written reason when downgrading a label (e.g. Confidential → General).

6 DLP policies created

Both policies start in simulation (TestWithoutNotifications) with Purview's native "Turn the policy on if it's not edited within fifteen days of simulation" flag set. Purview itself owns the countdown; the toolkit does not auto-enforce.

Exchange — block Confidential / HC external Default

Simulation SMBTool - DLP - Confidential and HC external (EXO)
  • Matches every Confidential and Highly Confidential sub-label.
  • Blocks the message when any recipient is outside the organisation.
  • Notifies the sender (SiteAdmin, LastModifier, Owner roles).
  • Incident report goes to SiteAdmin.

SharePoint + OneDrive — block external share Default

Simulation SMBTool - DLP - Confidential and HC external (SPO+ODB)
  • Same six labels matched.
  • Blocks external sharing only (internal collaboration unaffected).
  • Notifies SiteAdmin, LastModifier, Owner.

Endpoint DLP — audit copy / print / USB E5 / Purview Suite

Simulation SMBTool - DLP - Endpoint Confidential and HC
  • Audits Copy/Paste, Print, RemovableMedia and NetworkShare actions on managed devices.
  • Flip individual entries to Block / Warn / BlockOverride after validating in DLP Activity Explorer.
  • Rejected automatically when the script runs with -BPOnly.

7 Retention — opt-in only

Exchange 7-year retain-then-delete -ApplyRetention Destructive

SMBTool - Exchange 7-year retention

Retains every mailbox item for 7 years from creation, then permanently deletes it tenant-wide. Off by default — you must pass -ApplyRetention. The 7-year default is wrong for some regulated verticals (law, accounting, healthcare, financial advisors, construction, real estate); confirm against the customer's record-keeping obligations before enabling.

See docs/Retention-Default-Risk.md for the decision matrix.

8 AI governance — auto-detected by licence

Block Microsoft 365 Copilot from Highly Confidential content Auto-detected

SMBTool - AI - Block Copilot for Highly Confidential · aligns to AI_054

Prevents Microsoft 365 Copilot (paid & free Copilot Chat) from grounding, summarising or returning search results from any document or email labelled Highly Confidential.

  • On automatically when the tenant has Microsoft 365 E5 or a Purview Suite SKU (detected via Graph /subscribedSkus).
  • Skipped automatically on Business Premium tenants (-BPOnly), because the Copilot DLP location is not licensed there.
  • Opt out explicitly with -SkipAIControls on any tenant.

9 Safety guarantees — how to re-run without breaking the tenant

Idempotent Re-runnable

The script can be re-run any number of times. Existing labels, policies and rules created by the toolkit are updated in place, not duplicated.

Ownership marker Tagged

Every object the toolkit creates carries [Managed by SMBTool Purview Toolkit] in its description. Untagged objects are left alone unless you pass -AdoptExisting.

Tenant-identity guard Pre-flight

After sign-in the script checks the connected tenant's verified domains against the expected domain (from -DelegatedOrganization or your admin UPN suffix). On mismatch the script aborts before any write.

Simulation mode DLP only

DLP policies are created in TestWithoutNotifications by default. Validate in DLP Activity Explorer first, then let Purview's 15-day flag flip them to enforce — or set DlpStartInSimulation = $false in the config to enforce immediately.

HTML & JSON report Every run

Each run writes Deploy-PurviewBestPractice-Report-<timestamp>.html and a machine-readable .json sidecar next to the script. Forward the HTML to the customer / project manager as evidence.

Secrets stripped Always

The report HTML-encodes every value and strips any parameter whose name matches password / token / secret / credential / key, so logs are safe to email.

i
Label and policy changes can take up to 24 hours to fully propagate across Microsoft 365 workloads. Always pilot in a test tenant before production rollout.
